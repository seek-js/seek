# Seek.js Docs

Public docs only. Keep ultra-minimal.

## Scope

Use `docs/` for:

- quick starts
- install/setup guides
- usage guides and tutorials
- integration guides
- API reference intended for framework users

Do not use `docs/` for:

- internal rules/policies
- architecture contracts (`specs/`)
- research (`research/`)

Internal documentation governance lives in root `README.md`.
